# Dev Session — 2026-03-13

## Priority Task: Group Attack Visual Feedback (Missing Signal Wire)

### Problem
Group attacks (All-Out Attack + Limit Break) were fully implemented in backend
and menu UI but had **zero visual feedback** — BattleScene never connected the
`group_attack_executing` signal emitted by BattleManager.

Players would see the battle log text but no party animations, no screen flash.
The feature was functionally invisible.

### Fix: `_on_group_attack_executing` handler in BattleScene.gd

**Signal wiring** (3 changes):
1. `_ready()`: connect `BattleManager.group_attack_executing → _on_group_attack_executing`
2. `_exit_tree()`: disconnect to prevent memory leaks (consistent with all other signals)
3. New handler: `_on_group_attack_executing(participants, group_type, targets)`

**Handler behavior:**
- **Screen flash**: Full-battlefield ColorRect tween fade
  - Limit Break → gold (`Color(1.0, 0.85, 0.0, 0.55)`) — 0.45s fade
  - All-Out Attack → orange (`Color(1.0, 0.5, 0.0, 0.40)`) — 0.45s fade
- **Simultaneous party animations**: iterates all participants, plays animation per type:
  - Limit Break → each member lunges toward closest enemy (`_animate_melee_attack`)
  - All-Out Attack → each member plays in-place `play_attack()` animation
- Skips dead participants and invalid indices safely

**Bug caught during implementation**: `play_attack(null)` fails type check — Godot
requires `Callable`, not null. Fixed to `play_attack()` (uses default empty Callable).
